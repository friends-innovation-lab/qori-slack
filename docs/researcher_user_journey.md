# 🧭 CivicMind Researcher Journey (GitHub Documentation)

This guide outlines the complete end-to-end experience for a civic tech researcher using CivicMind to manage a human-centered research study.

---

## 👩🏽‍💻 Researcher Journey Overview

### 1. Project Kickoff
- Create a new project in Slack: `/civicmind new_project "Benefits Navigation"`
- CivicMind generates:
  - Research plan (markdown)
  - Consent form template
  - Participant tracker

### 2. Upload & Process Research Artifacts
- Upload audio: CivicMind transcribes and tags automatically
- Upload docs: CivicMind summarizes PDFs, Google Docs, or Markdown

### 3. Interactive Research Q&A
- Ask via Slack: `/civicmind ask "What did users say about eligibility?"`
- CivicMind responds with:
  - Summary of insights
  - Supporting quotes
  - Source links
  - Related tags

### 4. Review & Approve AI Outputs
- Researcher reviews AI-generated content:
  - Markdown research summary
  - Insight cards
  - Can edit, approve, reject

### 5. Share with Stakeholders
- Generate formatted readouts:
  - Markdown for GitHub
  - Slack posts
  - PDF or slide exports

### 6. Convert to Tasks
- Turn approved insights into:
  - GitHub Issues
  - Jira tasks with labels and acceptance criteria

### 7. Weekly Digest
- CivicMind auto-generates:
  - Weekly insight roundup
  - Progress snapshot
  - Pushes to `#research-digests`

## 📄 How Insight Cards Work

Each tagged quote or theme becomes a **Markdown card** formatted like:

```markdown
🧠 **Insight:** Users are unclear about eligibility

**Theme:** Eligibility / Clarity

**Quote:**  
> "I wasn’t sure if I even qualified."

**Suggested Action:**  
Add a checklist or plain-language eligibility page.

**Source:** Interview_01.txt  
**Tags:** `benefits`, `confusion`  
**Reviewed by:** @researcher-name
```

These cards can:
- Be posted in Slack
- Used to auto-create GitHub Issues
- Grouped into thematic clusters

## 📄 How Research Summaries Work

Markdown summaries follow this structure:

```markdown
# CivicMind Research Summary: Benefits Navigation
_Prepared by: CivicMind (AI) + @researcher_

## 🔍 Key Themes
- Eligibility confusion
- Overwhelming form layout
- Mismatch between expectations and experience

## 🧠 Representative Quotes
> "The form is just a wall of text."  
> "I had no idea which documents I needed."

## 🧩 Recommendations
- Add visual progress indicators
- Introduce document checklist step

## 📁 Source Files
- Interview_01.txt
- Benefits_policy_summary.md
```

## 📑 Adapting VA Standard Templates

We will support:
- VA-style research plan templates (Markdown, GitHub-native)
- VA standard readout templates
- Auto-generation of plan.md and readout.md using a base template filled by Slack prompts or UI inputs

Example:
- `/civicmind plan benefits-navigation`
- CivicMind generates `research-plan.md` with standard sections filled in

## 🔒 Human in the Loop

- Labeled as “AI-generated”
- Editable before sharing
- Stored with version history and reviewer metadata
