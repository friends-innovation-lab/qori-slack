# Contributing to CivicMind

We welcome contributions from civic tech designers, developers, researchers, and public service practitioners.

Whether you're fixing bugs, improving documentation, or building new features, your work helps improve civic research for everyone.

## 🛠️ How to Contribute

1. **Fork the repository**
2. **Create a feature branch**
git checkout -b feature/your-feature-name
3. **Commit changes with clear messages**
4. **Push and submit a pull request**
5. Add context to your PR: what, why, and how to test it.

## 🧪 Good First Issues

Check out [our open issues](https://github.com/your-org/civicmind/issues) for ideas—especially those labeled `good first issue`.

## 📚 Development Setup

- Python 3.10+
- Node.js (for Slackbot, optional)
- Postgres (locally or hosted)
- Recommended: `pipenv` or `poetry`

## 🧠 Design Guidelines

- AI output should always be *editable*, *auditable*, and *clearly marked as AI-generated*.
- Respect user privacy: no unapproved personal data should be committed.
- Make interfaces accessible (WCAG 2.1 AA).

## 💬 Questions?

Open an issue or join the CivicMind Slack workspace to get feedback and collaborate.
