# User Testing Session Notes - PT003 Marcus Johnson
**UX Researcher:** Alex Chen  
**Date:** August 14, 2025  
**Time:** 10:00-11:00 AM EST  
**Research Focus:** User journey, emotional responses, task completion patterns

## Session Overview
Veteran user (Marcus, SFC retired) testing VA mobile app. Rural Alabama, uses accessibility features. Generally patient but has clear frustration points with performance.

## User Behavior Patterns

### Initial Reactions
- **Minute 2:** Immediately noticed and appreciated large text - "Actually, this looks pretty good"
- **Minute 3:** Scanning behavior efficient - quickly identified main navigation
- **Minute 4:** Positive comment about button clarity and organization

### Frustration Triggers
- **Minute 6:** First loading delay comment - tone shift noticeable
- **Minute 12:** Multiple sighs during load times - body language change
- **Minute 18:** Started tapping screen impatiently 
- **Minute 22:** Verbalized considering alternatives: "easier to just talk to a person"

### Adaptation Behaviors
- Uses accessibility features effectively
- Has backup strategies (phone calls)
- Patient initial approach, escalates predictably
- Provides constructive feedback even when frustrated

## Task Analysis

### Task 1: Payment History (8 minutes)
- **Mental Model:** Expected "payments" or "money" section
- **Navigation:** Direct path to correct area
- **Blocker:** Loading times only - no usability issues
- **Completion:** Successful but lengthy due to performance

### Task 2: Update Phone Number (6 minutes)  
- **Mental Model:** Looked in "Profile" first (correct)
- **Pain Point:** Edit button too small despite large text
- **Workaround:** Used zoom gesture to locate button
- **Completion:** Successful with minor accessibility barrier

### Task 3: Claim Status Check (FAILED - 10 minutes)
- **Mental Model:** Expected historical claims in main claims area
- **Confusion Point:** "No active claims" message unclear
- **Search Behavior:** Checked multiple sections before giving up
- **Abandonment:** Clear stopping point when couldn't find info

### Task 4: Messaging (7 minutes)
- **Expectation:** Anticipated complexity based on previous tasks
- **Pleasant Surprise:** Actually praised interface once loaded
- **Learning:** Adapted to slow loading by task 4

## Emotional Journey Map

**Start:** Optimistic (large text working well)

**Middle:** Frustration peak (claim status failure)  
**Late Middle:** Resigned acceptance (adjusted expectations)  
**End:** Constructive (offering specific suggestions)

## Key Quotes - Emotional Context

**Delight:**
> "That's better than a lot of apps I use" - genuine surprise at accessibility

**Frustration:**
> "This is pretty typical for this app" - resigned familiarity with poor performance

**Problem-Solving:**
> "Where would a completed claim show up?" - logical troubleshooting attempt

**Patience Limit:**
> "By the time this app loads what I need, I could have talked to a person" - clear breaking point

## User Needs Identified

### Primary Needs
1. **Speed/Performance:** Critical for task completion
2. **Information Findability:** Claims status particularly important
3. **Accessibility:** Large text working but needs consistency
4. **Reliability:** Backup options essential for this user

### Unmet Needs
- Historical information access
- Progress indicators for slow connections  
- Help/support within app context
- Consistent accessibility across all interface elements

## Design Recommendations

### Immediate
- Optimize for low-bandwidth connections
- Add better loading states/progress indicators
- Ensure all interactive elements meet accessibility size requirements

### Medium-term  
- Add offline/cached content for basic info
- Improve information architecture for claim status
- Integrate help/chat functionality

### Long-term
- Design specifically for rural connectivity patterns
- Consider progressive web app features for better performance

## Notable Observations
- User has realistic expectations based on past experience
- Maintains respect and constructive attitude despite frustrations
- Successfully uses workarounds when needed
- Values human backup options for complex issues
- Large text accessibility generally working well - good implementation

**Overall Assessment:** Experienced user with clear needs who adapts well to limitations but has identifiable breaking points around performance and information access.
